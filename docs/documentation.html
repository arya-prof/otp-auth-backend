Project Overview

A robust backend service for OTP-based authentication and user management built with Go, featuring secure JWT tokens, Redis caching, PostgreSQL database, and comprehensive API documentation.

System Architecture

System architecture showing the flow from client requests through services to data storage

Backend API

Go/Gin server with JWT authentication and OTP verification

Database

PostgreSQL with automatic migrations and connection pooling

Caching

Redis for OTP storage and rate limiting

Key Features

  • OTP Authentication: Secure phone number verification with rate limiting
  • JWT Tokens: Stateless authentication with configurable expiration
  • User Management: CRUD operations with pagination and search
  • Rate Limiting: Configurable request throttling for security
  • API Documentation: Auto-generated Swagger/OpenAPI specs

Setup Guide

Quick Start

Follow these steps to get the OTP Authentication Backend running:

1. Clone and Setup

git clone https://github.com/arya-prof/otp-auth-backend.git
cd otp-auth-backend
go mod tidy

2. Start with Docker

docker-compose up -d

# Or build and run locally
go run cmd/main.go

3. Access Services

  • • Backend API: http://localhost:8080
  • • Swagger UI: http://localhost:8080/swagger/index.html
  • • Health Check: http://localhost:8080/health

API Reference

Base URL

All API endpoints are prefixed with: http://localhost:8080/api/v1

Authentication Endpoints

POST /api/v1/auth/request-otp

Generate and send OTP to the specified phone number

curl -X POST http://localhost:8080/api/v1/auth/request-otp \
-H "Content-Type: application/json" \
-d '{"phone": "+1234567890"}'
POST /api/v1/auth/verify-otp

Verify OTP code and authenticate user

curl -X POST http://localhost:8080/api/v1/auth/verify-otp \
-H "Content-Type: application/json" \
-d '{"phone": "+1234567890", "otp": "123456"}'

User Management Endpoints

GET /api/v1/users

Get paginated list of users (requires authentication)

curl -X GET "http://localhost:8080/api/v1/users?page=1&limit=10" \
-H "Authorization: Bearer YOUR_JWT_TOKEN"
GET /api/v1/users/{id}

Get user by ID (requires authentication)

curl -X GET "http://localhost:8080/api/v1/users/user-uuid" \
-H "Authorization: Bearer YOUR_JWT_TOKEN"

Database Choice Justification

PostgreSQL - Primary Database

  • • ACID compliance for data integrity
  • • Advanced indexing and query optimization
  • • JSON support for flexible data structures
  • • Robust user management and permissions
  • • Excellent Go driver support (lib/pq)

Redis - Caching Layer

  • • Sub-millisecond response times
  • • Built-in TTL for OTP expiration
  • • Atomic operations for rate limiting
  • • Pub/Sub for real-time features
  • • Memory-efficient data structures

Architecture Benefits

Data Persistence PostgreSQL ensures user data survives restarts
Performance Redis provides lightning-fast OTP lookups
Scalability Horizontal scaling with read replicas
Security Role-based access control and encryption

Use Case Alignment

Perfect for OTP Authentication

This combination excels for OTP systems: PostgreSQL stores user profiles and authentication history, while Redis handles temporary OTP codes with automatic expiration and rate limiting.

Docker Setup

Containerized Deployment

The project includes Docker configuration for easy deployment and development.

1. Build and Start Services

# Build and start all services
docker-compose up -d

# View logs
docker-compose logs -f app

2. Service Management

# Stop services
docker-compose down

# Rebuild after code changes
docker-compose build --no-cache

3. Service Access

  • • Backend API: http://localhost:8080
  • • PostgreSQL: localhost:5432
  • • Redis: localhost:6379
  • • Swagger UI: http://localhost:8080/swagger/index.html
